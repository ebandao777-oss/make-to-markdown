# Make-to-Markdown — 使用说明

> 工业级 RAG Markdown 物料生成器。将 20+ 种文档格式一键转换为清洗后的 Markdown，为知识库提供统一的"通用语言"。

---

## 1. 功能概览

本工具负责将任意文档转换为 RAG 就绪的 Markdown，核心流程全自动：**依赖检测 → 自动安装 → 格式转换 → 失败降级 → 后置清洗**。

### 支持的输入格式

| 类别 | 格式 |
|:---|:---|
| 文档 | `.pdf` `.docx` `.doc` `.epub` |
| 表格 | `.xlsx` `.xls` `.csv` |
| 演示 | `.pptx` `.ppt` |
| 网页 | `.html` `.htm` |
| 数据 | `.json` `.xml` |
| 图片 | `.png` `.jpg` `.jpeg` `.gif` `.bmp` `.tiff` `.webp`（含 EXIF/OCR） |
| 音频 | `.mp3` `.wav` `.m4a` `.ogg` `.flac`（含语音转写） |
| 压缩包 | `.zip` |
| 视频 | YouTube 链接 |

### 核心能力

- **零手工依赖**：自动检测并安装所需 Python 包，无需手动 `pip install`
- **旧格式兼容**：`.doc` / `.ppt` 自动通过 Office COM 或 LibreOffice 转为新版格式
- **智能降级**：markitdown 失败时自动切换原生库（python-docx / openpyxl / python-pptx）
- **后置清洗**：自动去除水印、页码、机密标记，修复标题层级，补全表格分隔符
- **批量转换**：递归遍历目录，保持原始结构，失败文件独立记录

## 2. 环境要求

| 依赖 | 说明 |
|:---|:---|
| Python 3.8+ | 系统内置或手动安装 |
| uv | markitdown 运行环境，[安装指南](https://docs.astral.sh/uv/) |
| Microsoft Office（可选） | .doc / .ppt 旧格式预处理（Windows） |
| LibreOffice（可选） | .doc / .ppt 旧格式预处理（全平台） |

首次运行时工具自动通过 `uv tool install markitdown --with=...` 安装 markitdown 及完整可选依赖。

## 3. 快速开始

### 单文件转换

```bash
cd path/to/Make-to-Markdown

# 转换 Word 文档
python scripts/convert.py report.docx -o report.md

# 转换 PDF
python scripts/convert.py paper.pdf

# 静默模式 + 跳过摘要
python scripts/convert.py data.xlsx -q --no-summary
```

### 批量转换

```bash
# 递归转换整个文件夹
python scripts/batch_convert.py ./source/ ./output/

# 仅转换指定格式
python scripts/batch_convert.py docs/ output/ --ext .pdf .docx
```

### 单独清洗已有 Markdown

```bash
python scripts/post_clean.py 文件.md --summary -o 清洗后.md
```

## 4. 脚本体系

| 脚本 | 用途 | 常用参数 |
|:---|:---|:---|
| `scripts/convert.py` | 智能转换引擎（推荐） | `-o <输出>` `-q` `--no-summary` |
| `scripts/batch_convert.py` | 批量转换 | `--ext` `--clean` |
| `scripts/post_clean.py` | 后置清洗 | `-o` `--summary` `--check-only` |
| `scripts/platform_detect.py` | 平台能力检测 | — |

### convert.py 工作流

```
输入文件 → 格式判断
  ├─ .doc/.ppt → COM/LibreOffice 预处理 → .docx/.pptx
  └─ 其他格式 → 直接进入
      ↓
依赖检测 → 自动安装缺失包 → markitdown 转换
      ↓ (失败)
原生降级转换器 (python-docx / openpyxl / python-pptx)
      ↓
后置清洗（去水印 / 标题修复 / 表格补全 / 摘要注入）
      ↓
输出 .md 文件
```

## 5. 常见问题

**Q: 提示 markitdown 依赖缺失？**
A: convert.py 启动时自动安装，通常无需手动干预。若网络受限，手动执行：
```bash
uv tool install markitdown --with python-docx --with openpyxl --with python-pptx --with pypdf --with xlrd --with ebooklib
```

**Q: .doc 文件转换失败？**
A: 需要安装 Microsoft Office（含 Word）或 LibreOffice。二选一即可。若两者都不装，脚本会暂停并提示。

**Q: 转换结果标题层级混乱？**
A: 后置清洗会自动修复（多 H1 降级、越级补过渡），无需手动处理。

**Q: 输出文件有水印残留？**
A: 清洗管线覆盖 15+ 噪声模式（Generated by markitdown、Confidential、Page X of Y 等），若仍有残留可单独运行 `post_clean.py`。

**Q: 加密文档如何处理？**
A: 工具会检测到加密提示，立即终止该文件并告知用户先解密。不会反复重试。

**Q: 批量转换时个别文件失败？**
A: 失败文件会被记录到 `输出目录/_conversion_errors.log`，其余文件正常转换，不会中断整个流程。

---

## 6. 输出质量验收

转换完成后建议检查以下 5 项：

1. 文件非空（> 0 字节）
2. 无乱码（UTF-8 编码正常）
3. 无水印/页码残留
4. 标题层级无跳跃（H1 → H3 不能没有 H2）
5. 表格分隔符完整

工具在转换完成时会打印 H1/H2/H3 统计和表格计数，可据此快速判断质量。
